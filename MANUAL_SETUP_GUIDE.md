# Полное руководство по ручной настройке RPG-Tracker

В этом документе собраны все действия, которые вам необходимо выполнить вручную в сторонних сервисах (Firebase, Apple Developer, App Store Connect), чтобы приложение работало корректно в продакшене. Я (ИИ) автоматизировал большую часть настроек Xcode, так что вам осталось сделать всего несколько кликов в консоли!

---

## 1. Авторизация (Apple & Google)

Я уже добавил **Capability "Sign in with Apple"** и прописал **URL Schemes для Google** в ваш проект Xcode. Вам нужно только включить их на сервере:

### Firebase Console
1. Перейдите в [Firebase Console](https://console.firebase.google.com).
2. Выберите проект `serzhanovich-ecosystem-ce700`.
3. Раздел **Authentication -> Sign-in method**:
   - Включите провайдер **Apple** (дополнительных ключей для iOS не нужно).
   - Включите провайдер **Google** (выберите Project support email).

---

## 2. База данных (Firestore) и Индексы

Для корректной работы таблиц лидеров (Leaderboards), Firebase требует создания составных индексов (Composite Indexes).
Я уже написал нужные конфигурации в файл `firestore.indexes.json`.

Вам **не нужно** делать это вручную! Просто откройте терминал в папке проекта и выполните:
```bash
firebase deploy --only firestore:indexes
```

---

## 3. Remote Config

Ваше приложение проверяет минимальную и рекомендуемую версии iOS для системы "Force Update".
1. В Firebase Console перейдите в **Engage -> Remote Config**.
2. Убедитесь, что там присутствуют и опубликованы (Publish) ключи:
   - `rpg_minimum_ios_version` (Например: `1.0.0`)
   - `rpg_recommended_ios_version` (Например: `1.0.0`)
